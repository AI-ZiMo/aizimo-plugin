# App Store Submission Guidelines

## Character Limits

### Promotional Text
- Maximum: 170 characters
- Can be updated anytime without new app submission
- Appears above the description on iOS 11+, macOS 10.13+
- Use to highlight current features or updates

### Description
- Maximum: 4,000 characters
- Should be compelling and informative
- Requires new app submission to update
- First few lines are most important (visible without "more" tap)

### Keywords
- Maximum: 100 characters
- Separate with English comma, Chinese comma, or mix of both
- No spaces needed between keywords
- Use singular and plural forms strategically
- Avoid duplicating words already in app name/company name

### App Name
- Maximum: 30 characters
- Appears in App Store and device home screen

### Subtitle
- Maximum: 30 characters
- Appears below app name in App Store

## Content Guidelines

### Do NOT Include

- Emojis in promotional text and description (may cause display issues)
- Special quotation marks like 「」 (use standard quotes)
- Special bullets like •, ✓ (use hyphens or numbers instead)
- Pricing information (use in-app purchase details instead)
- Competitive app references
- Placeholder or lorem ipsum text
- Links to external websites (except privacy policy/terms of service)

### Do Include

- Clear value proposition in first paragraph
- Key features with benefits
- Privacy and data handling information
- Support for multiple languages if applicable
- Accessibility features if applicable
- Unique selling points that differentiate from competitors

## Best Practices

### Promotional Text
- Lead with strongest feature or current update
- Create urgency or highlight exclusivity
- Keep it concise and scannable
- Test in both languages if multilingual

### Description
- Hook readers in first 2-3 sentences
- Use clear structure: Introduction → Features → Benefits → Call-to-action
- Break content into scannable sections
- Highlight privacy features (increasingly important to users)
- Include target audience information
- Avoid overly technical jargon unless targeting technical users

### Keywords
- Research competitor keywords
- Use app-specific terms
- Include category-relevant keywords
- Avoid keyword stuffing
- Prioritize most important keywords first (order matters)
- Use variations of main keywords
- Include both English and localized keywords for international markets

## Localization Tips

### Chinese Submissions
- Use simplified Chinese for China App Store
- Avoid overly literal translations
- Consider cultural context and preferences
- Privacy is a major concern - emphasize local storage and no data collection
- Use concise, impactful language
- Gaming/gamification concepts resonate well with Chinese users

### English Submissions
- Use American English spelling for US App Store
- Be direct and benefit-focused
- Emphasize convenience and time-saving
- Privacy and security are strong selling points
- Use active voice

## Common Mistakes to Avoid

1. Exceeding character limits (app will be rejected)
2. Including emoji or special characters in wrong fields
3. Generic descriptions that could apply to any app
4. Keyword stuffing or repetition
5. Not updating promotional text to reflect new features
6. Ignoring first-impression content (first 3 lines of description)
7. Not A/B testing different descriptions
8. Forgetting to localize keywords
9. Using technical jargon for non-technical audiences
10. Not highlighting privacy/security features
